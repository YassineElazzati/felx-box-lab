Un carré qui voyage

Découvre display: flex et justify-content

Analogie pour le HTML

Imagine le HTML comme un plan d'architecte. Une balise <div> est une boîte vide qu'on dessine sur le plan. Une boîte peut contenir d'autres boîtes : c'est le principe des poupées russes.

L'attribut class="..." est comme une étiquette qu'on colle sur la boîte. Plus tard, en CSS, on dira "applique ce style à toutes les boîtes qui portent l'étiquette X".

Ici, on a une grande boîte étiquetée demo, et à l'intérieur, une petite boîte étiquetée carre.

Le concept Flexbox

Quand tu écris display: flex sur une boîte, elle devient un conteneur flex. Tous ses enfants directs deviennent des flex items et se rangent automatiquement sur une ligne horizontale.

La propriété justify-content contrôle la position des items sur l'axe principal (par défaut horizontal, de gauche à droite).

Analogie pour le CSS

Imagine que la boîte demo est un plateau de service dans un restaurant. Tu poses des verres dessus.

▸ flex-start : tu pousses tous les verres contre le bord gauche du plateau
▸ center : tu rassembles les verres au milieu du plateau
▸ flex-end : tu pousses tous les verres contre le bord droit

Mission

Envoie le carré à gauche, puis au centre, puis à droite. Lis le CSS qui se met à jour à chaque changement.

A
HTML
<div class="demo">
  <div class="carre">A</div>
</div>
CSS
.demo {
  display: flex;
  justify-content: flex-start;
}

.carre {
  width: 80px;
  height: 80px;
}
2

Plusieurs carrés et de l'espace

Distribue l'espace avec space-* et gap

Analogie pour le HTML

Maintenant on pose trois verres identiques sur le même plateau. En HTML, ça veut juste dire qu'on écrit trois fois la même balise <div class="carre"> à l'intérieur du conteneur.

Note bien : les enfants se mettent les uns à la suite des autres, dans l'ordre où on les écrit dans le HTML.

Le concept Flexbox

Avec plusieurs items, justify-content ne sert plus seulement à les pousser d'un côté : il peut aussi distribuer l'espace entre eux.

▸ space-between : premier item collé à gauche, dernier item collé à droite, espace réparti entre les items. Aucun espace sur les bords.
▸ space-around : chaque item a un espace identique autour de lui. Les bords ont la moitié de l'espace par rapport à l'intérieur.
▸ space-evenly : tous les espaces sont strictement égaux, y compris ceux des bords.

La propriété gap est différente : elle ajoute un écart fixe entre les items, sans toucher aux bords. On peut combiner gap avec justify-content.

Analogie pour le CSS

Reprends le plateau de service avec trois verres.

▸ space-between = une rangée d'élèves alignés contre les murs d'une salle de classe : le premier touche le mur gauche, le dernier touche le mur droit, et les autres se répartissent au milieu.
▸ space-around = des piliers d'autoroute espacés régulièrement, chacun avec sa propre "zone perso" autour de lui.
▸ gap = la marge entre deux pots de fleurs que tu fixes toi-même à l'avance, indépendamment de l'espace total disponible.

Mission

Teste space-between puis space-around. Observe attentivement les bords du conteneur. Que vois-tu ?

Astuce

Ajoute aussi du gap et regarde comment ça se combine (ou pas) avec les valeurs space-*.

C
HTML
<div class="demo">
  <div class="carre">A</div>
  <div class="carre">B</div>
  <div class="carre">C</div>
</div>

}
3

L'axe vertical entre en jeu

Maîtrise align-items sur l'axe secondaire

Analogie pour le HTML

La structure HTML est exactement la même qu'avant : trois carrés dans une boîte. Le HTML décrit QUI est dans la page, le CSS décide OÙ et COMMENT les choses sont placées.

Ce qui change ici, c'est qu'on a augmenté la hauteur du conteneur (320px au lieu de 200px) pour avoir de la place verticalement et pouvoir jouer avec le second axe.

Le concept Flexbox

Flexbox travaille toujours avec deux axes :

▸ L'axe principal : celui sur lequel les items sont alignés (par défaut horizontal). On le contrôle avec justify-content.
▸ L'axe secondaire : perpendiculaire à l'axe principal (par défaut vertical). On le contrôle avec align-items.

Valeurs de align-items :

▸ stretch (par défaut) : les items s'étirent pour remplir toute la hauteur, SAUF s'ils ont une height fixée
▸ flex-start : items collés en haut
▸ center : items centrés verticalement
▸ flex-end : items collés en bas

Le combo magique pour centrer parfaitement : justify-content: center + align-items: center. C'est LA technique la plus utilisée en CSS moderne.

Analogie pour le CSS

Pense à un tableau blanc dans une salle de classe sur lequel tu colles trois post-it.

▸ justify-content décide où horizontalement tu places la rangée (à gauche, au milieu, à droite du tableau).
▸ align-items décide où verticalement tu colles cette rangée (en haut du tableau, au milieu, en bas).

Combinées, ces deux propriétés agissent comme des coordonnées GPS : horizontal + vertical = position exacte sur le tableau.

Missions

1 — Centre les 3 carrés au milieu absolu du conteneur.
2 — Colle-les en bas à droite.
3 — Répartis-les horizontalement avec space-between, tous centrés verticalement.

A
B
C
HTML
<div class="demo">
  <div class="carre">A</div>
  <div class="carre">B</div>
  <div class="carre">C</div>
</div>
CSS
.demo {
  display: flex;
  height: 320px;
  justify-content: flex-start;
  align-items: stretch;
}
4

Pivoter la direction

Avec flex-direction, l'axe principal change

Analogie pour le HTML

Toujours la même structure HTML : trois <div> dans un parent. Tu vois maintenant pourquoi le HTML est important : il ne change presque jamais. C'est seulement le CSS qui transforme l'apparence visuelle.

C'est la séparation contenu / présentation : le HTML décrit le sens des choses, le CSS s'occupe du look.

Le concept Flexbox

Par défaut, les items s'alignent horizontalement (flex-direction: row). Mais on peut pivoter cet axe principal pour qu'il devienne vertical.

▸ row (par défaut) : axe principal = horizontal, gauche → droite
▸ row-reverse : axe principal = horizontal, droite → gauche
▸ column : axe principal = vertical, haut → bas
▸ column-reverse : axe principal = vertical, bas → haut

Le piège qui perd tout le monde : quand tu passes en column, l'axe principal devient vertical, donc justify-content contrôle maintenant le vertical, et align-items contrôle l'horizontal. Les deux propriétés s'inversent.

Les noms justify-content et align-items ne sont pas liés à "horizontal" ou "vertical", ils sont liés aux notions d'axe principal et d'axe secondaire.

Analogie pour le CSS

Imagine que tu fais tourner un livre posé à plat sur une table d'un quart de tour. Ce qui était la "longueur" devient la "largeur", et inversement.

C'est exactement ce que fait flex-direction: column : l'axe principal pivote. Du coup, la propriété qui déplaçait tes carrés de gauche à droite les déplace maintenant de haut en bas. Le mot justify-content n'a pas changé, mais sa direction d'action oui.

Autre image : pense à une boussole. justify-content pointe toujours dans la direction principale, mais cette direction peut être Nord-Sud ou Est-Ouest selon comment tu orientes la boussole.

Missions

1 — Passe en column et empile les 3 carrés verticalement, centrés horizontalement.
2 — En column, mets les carrés en bas du conteneur.
3 — Teste row-reverse. Quel carré est maintenant à gauche ? Pourquoi ?

Test de compréhension

En mode column, si tu veux centrer un seul carré au milieu absolu, tu utilises quelles propriétés ?

Réponse : exactement les mêmes qu'en mode row ! Le combo justify-content: center + align-items: center marche dans toutes les directions, parce qu'il centre sur les deux axes à la fois.
